# KOMIPO Safety System: Core Development Rules (v35.8.2)

사용자님과 수십 차례의 반복 개선을 통해 정립된 **KOMIPO 스마트 안전점검 시스템**의 철학과 기술적 규칙을 정리한 최종 가이드입니다.

---

### 1. UI 및 사용자 경험 (UI/UX)
- **독립적 배너 시스템**: 상단 배너(부서명 > 작업명)는 스테퍼(Stepper)의 노출 여부와 상관없이 항상 최상위에 독립적으로 존재해야 함.
- **단계별 노출 규칙**:
    - **부서 선택 시**: 배너 숨김.
    - **작업 선택 시**: **[부서명]**만 배너에 표시.
    - **평가 및 이후 단계**: **[부서명] > [작업명]**을 배너에 모두 표시.
- **프리미엄 미학**: 카드 기반 레이아웃, Lucide 아이콘, 블루 계열의 정제된 색상 사용.

### 2. 데이터 매칭 및 지능형 엔진 (Data matching)
- **매칭 우선순위 (getFuzzyPreInfo)**:
    1. **Exact Full Match**: 부서명 + 작업명이 CSV(pre_info_data.js)와 100% 일치할 때.
    2. **Exact Task Match**: 부서와 관계없이 작업명이 CSV와 100% 일치할 때.
    3. **Similarity Match**: 레벤슈타인 거리 기반 유사도 60% 이상일 때.
    4. **Keyword/Fallback**: 매칭 실패 시 기본 '표준 수공구/안전구' 가이드 출력.
- **데이터 브릿지**: `master_data.js`(평가 문항)와 `pre_info_data.js`(사전 정보)가 반드시 동시에 로드되어야 함.

### 3. 구글 시트 데이터 전송 (Data Integrity)
- **작업단계(Step) 완전 보존**: 평가 항목 상단에 표시되는 '작업단계(예: 작업준비)'는 반드시 시트에 기록되어야 함.
- **다중 키 전송(Redundancy)**: 구글 앱스 스크립트의 유연한 대응을 위해 `step`, `작업단계`, `work_step`, `phase` 라는 4개의 키를 동시에 전송함.
- **데이터 보강**: row 데이터에 작업단계가 없을 경우, 현재 선택된 `selectedTask`를 강제로 대입하여 빈칸 전송을 방지함.

### 4. 상태 관리 (State Management)
- **실시간 동기화**: `switchPhase` 호출 시 반드시 `currentState.currentPhase`를 먼저 갱신하고 `syncUIState`를 호출해야 함.
- **버전 관리**: 모든 업데이트 시 `index.html`과 `app.js`의 버전을 일치시키고 하단 마커를 갱신함.
