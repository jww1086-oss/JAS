# 🎨 KOMIPO 스마트 안전 UI/UX 상세 명세 (v1.0-REGISTRY)

본 문서는 안티그래비티(AI)가 대화의 맥락을 잃더라도, 이 명세만 보고 **가장 완벽했던 UI를 100% 재현**할 수 있도록 하는 '디자인 정답지'입니다. 모든 UI 수정 시 이 명세와 대조(Audit)해야 합니다.

---

## 1. 뼈대 및 핵심 토큰 (Visual Foundations)

### 1.1 기본 컬러 테마
- **Primary (Indigo-Blue)**: `linear-gradient(135deg, #1e293b 0%, #334155 100%)` (주요 카드 및 헤더)
- **Doing Blue**: `#0ea5e9` (진행 중인 상태, 버튼)
- **Success Green**: `#10b981` (완료 상태, 체크박스 활성화)
- **Danger Red**: `#ef4444` (조치 필요, 미이행 상태 강조)
- **Background**: `#f8fafc` (연한 그레이 톤의 고급스러운 배경)

### 1.2 공통 디자인 요소 (Common UI)
- **카드 스타일**: `background: white; border-radius: 24px; border: 1.5px solid #f1f5f9; box-shadow: 0 4px 10px rgba(0, 0, 0, 0.04);`
- **폰트**: `Inter` 또는 `Outfit` (600~950 weight 위주의 강한 대비)

---

## 2. 단계별 화면 구성 (Step-by-Step Spec)

### 📊 [Step 1] 부서 및 작업 선택
- **부서 선택**: 6개 부서 배너 나열. 파란색 상단 배너 숨김(Home) 또는 부서명 표시.
- **작업 선택**: 149개 작업 나열. 파란색 상단 배너에는 **부서명**만 표시 (동기화 엔진에 의해 작업명 강제 숨김).
- **평가 단계**: 파란색 상단 배너에 **부서명 > 작업명** 모두 표시.



### 👥 [Step 1.5] 평가 참여자 및 사전 정보 (v35.7.1)
- **평가자 입력**: 칩(Tag) 스타일의 멀티 입력 인터페이스.
- **사전 정보 카드**: 지능형 Fuzzy 검색 엔진(v35.7.0) 탑재. `타 부서 참조`, `유사 작업 참조` 배지 활용.
- **Vibe**: 점검 품질을 높이기 위한 "인지(Awareness)" 단계로, 정보가 눈에 잘 들어오도록 풍부한 아이콘 사용.

### ✅ [Step 2] 위험 요인 체크리스트
- **위험요인 카드**: `border-radius: 24px`, 클릭 시 아코디언 확장.
- **체크박스**: 좌측 배치, 파란색 테두리(`var(--doing-blue)`).
- **위험도 보드 (핵심)**:
  - **구성**: `[제목] | 빈도(L) [Select] | 강도(S) [Select] | R: [Value]`
  - **레이아웃**: `display: flex; align-items: center; justify-content: space-between; gap: 10px;` (한 줄 수평 정렬)

### 🚀 [Step 3] 개선 조치 및 사진 (Strict RED)
- **노출 조건(v35.6.1)**: Step 2에서 **빨간색 체크박스(RED)**를 선택한 항목만 생성됨. (단순 미체크 항목은 배제)
- **UI 요소**: 카드 상단 우측에 `RED CHECK 필수` 라벨 (배경: `#fef2f2`, 텍스트: `#ef4444`).
- **사진 인터페이스**: 카드 정중앙에 큰 **카메라 아이콘** 필수. 클릭 시 카메라 트리거.
- **빈 화면 처리**: 빨간색 체크가 없을 경우 "별점/빨간색 체크 없음" 안내와 함께 최종 리포트 이동 버튼 제공.

### ✍️ [Step 4] 결과 미리보기 및 서명 (Clean Report)
- **노출 필터링**: 사용자가 최소 하나라도 체크(Blue 또는 Red)한 항목만 최종 요약 리스트에 표시.
- **내용 구성**: 
  - **Blue Section**: 이행 완료 조치 텍스트만 표시. (미이행 placeholder 제거)
  - **Red Section**: 개선 대책 텍스트 + 사진 + 의견 결합 노출.
- **미리보기 리포트**: 다크 다이얼로그 스타일의 상단 요약 카드 필수 (`Indigo Gradient`).
- **서명 패드**: 캔버스는 투명 배경, `penColor: '#1e293b'`.
- **버전 표시**: 하단 푸터에 `vX.Y.Z | Strict Filtering Applied` 문구와 버전 정보 상시 노출.

## 3. 에이전트 행동 지침 (AI Code Audit Protocol)

1. **상태 보존**: UI를 수정할 때마다 `currentState`의 `Set` 객체들이 유실되지 않는지 반드시 체크할 것.
2. **구조 복사**: 새로운 카드를 생성할 때 위 명세의 CSS 속성을 그대로 복제해서 사용할 것 (`px` 단위, `border-radius` 등).
3. **가독성 사수**: 모바일 웹 뷰이므로 폰트 크기가 너무 작아지지 않도록 (`0.85rem` 이상 유지) 관리할 것.
