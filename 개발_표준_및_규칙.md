# 🏛 KOMIPO 스마트 안전 점검 시스템 - 기술 명세 및 개발 표준 (v2.0-ULTIMATE)

이 문서는 프로젝트의 지속 가능성, 데이터 무결성, 그리고 프리미엄 UX 일관성을 유지하기 위한 **상세 기술 명세이자 개발 헌법**입니다. 모든 개발자 및 AI 에이전트는 본 가이드를 한 자의 오차 없이 준수해야 합니다.

---

## 1. 🏗 시스템 아키텍처 및 기술 스택
### 1.1 핵심 기술 (Core Tech)
- **Frontend**: Vanilla JS (ES6+), HTML5, CSS3 (No Framework)
- **Backend API**: Google Apps Script (GAS) 웹 앱 배포 버전.
- **Data Source**: Google Sheets (위험성평가자료, 작업정보, 이력).
- **Offline Support**: `localStorage`를 활용한 데이터 영속성 보장.

### 1.2 외부 종속성 관리
- **Icon**: Lucide v0.263.0 (CDN 로드)
- **Signature**: SignaturePad.js (Canvas 기반)
- **Scroll**: Vanilla Smooth Scroll (CSS 인라인 처리)
- **Fonts**: Google Fonts (Inter, Outfit) 활용.

## 3. 💾 데이터 아키텍처 및 이원화 관리 규칙
### 3.1 마스터 데이터 (Master Data - 정적/오프라인)
- **소스**: PDF 내 '위험성평가자료' 시트 및 기존 `master_data_utf8.js`.
- **저장**: `master_data.js` 및 `master_data_extra.js`에 JavaScript 객체 형태로 내장.
- **목적**: 앱 수동 기동 및 오프라인 환경에서의 즉각적인 작업 선택과 위험성 평가 수행 보장.

### 3.2 트랜잭션 데이터 (Transaction Data - 실시간/온라인)
- **소스**: 구글 시트의 '위험성분석결과' 또는 '제출이력' 시트.
- **연동 방식**: '위험성평가 결과 조회' 화면 진입 시 `GAS_URL`을 통해 실시간으로 최신 20~50개의 이력을 fetch.
- **필수 규칙**: 모든 실시간 요청에는 `?t=${Date.now()}` 값을 첨부하여 캐시를 무력화하고, 데이터 로딩 중에는 반드시 Premium Loader(Section 2.2)를 표시함.

### 3.2 Google Sheets 동기화 규칙
- **JSONP 활용**: CORS 이슈 회피를 위해 `fetchJSONP` 또는 `no-cors` 모드의 `fetch` 필수.
- **캐시 차단**: 모든 요청 URL 끝에 `?t=${Date.now()}` 첨부.
- **수치 보정 로직**: 
  - 시트 데이터 중 빈도/강도가 비어있거나 `0`인 경우, 마스터 데이터에서 해당 문구와 일치하는 값을 찾아 주입함. 
  - 기본값은 `현재빈도: 1, 현재강도: 1`로 설정하여 위험도 계산 오류를 차단함.

## 5. 🛠 에이전트(AI) 작업 프로세스 및 에러 대응 수칙
### 5.1 기본 작업 수칙
1. **Context First**: 수정 전 파일의 전체 내용을 `view_file`로 확인하고, 기존의 버전 주석(예: `[v34.x]`)을 훼손하지 않는다.
2. **Atomic Edit**: 한 번에 너무 많은 파일을 수정하지 않고, 컴포넌트 단위로 분리하여 수정한다.
3. **Validation**: JavaScript 수정 후에는 가상의 `node -c` 체크 또는 로직 정합성을 다시 검토한다.
4. **Plan First**: 모든 비정형 작업은 `implementation_plan.md`를 먼저 업데이트하고 사용자의 명시적 승인을 받는다.

### 5.2 ⚠️ 에러 대응 프로토콜 (3-Fail Pivot Rule)
동일한 원인 혹은 유사한 유형의 에러가 **3번 연속 또는 반복적**으로 발생할 경우, 기존의 접근 방식을 즉시 중단하고 다음과 같이 **방법론을 변경(Pivot)**해야 합니다.
- **데이터 추출 실패 시**: 자동 파싱 중단 -> 원시 데이터 기반 수동 정제 또는 샘플 기반 점진적 확대로 전환.
- **코드 로직 결함 시**: 복잡한 병합 중단 -> 최소 기능 단위(MVP)부터 재구축하거나 안정적인 이전 버전으로 롤백 후 단계적 수정.
- **환경 이슈 발생 시**: 외부 라이브러리/API 의존성 제거 -> 순수 Vanilla 로직으로의 대체 시도.

## 6. 📑 PDF/CSV 마스터 데이터 연동 및 규격 (Data Integrity)
### 6.1 데이터 규모 절대 기준 (Absolute Scale)
모든 마스터 데이터 연동 시 다음 수치를 **절대적으로 준수**해야 하며, 이 범위를 벗어날 경우 데이터 결손 또는 에러로 간주합니다.
- **총 부서명 수**: **6개** (정확히 일치해야 함)
- **총 작업명 수**: **149개** (누락 없이 연동되어야 함)
- **총 위험성 평가 레코드 수**: **2,000개 이상** (csv 전수 데이터 포함)

### 6.2 데이터 연동 상세 규칙
- **항목 매칭**: PDF 추출 시 `부서명-작업명-작업단계-위험요인-현재안전조치-개선대책` 순서를 유지하며, 149개의 작업명이 `EXTRA_INFO` 리스트에 트리 구조로 완벽히 매핑되어야 한다.
- **무결성 검사**: 연동 후 고유 작업명 개수가 149개가 아닐 경우, 에러 대응 프로토콜(3-Fail Pivot Rule)에 따라 추출 방식을 즉시 변경해야 한다.

## 7. 🔄 데이터 클렌징 및 UI 인터랙션 특화 규칙 (v35.4.2 추가)
### 7.1 데이터 무결성 및 인코딩 (Data Sanitization)
- **NFC 정규화**: 모든 한글 데이터는 저장/로딩 시 `normalize('NFC')`를 거쳐 자음/모음 분리 및 깨짐 현상을 방지해야 함.
- **지능형 키 탐색 (Intelligent Mapping)**: 소스 데이터(CSV/JSON)의 컬럼명이 오염되거나 물음표(`?`)가 섞여 있어도, 정규표현식 매칭을 통해 표준 키(`부서명`, `작업명`, `작업단계`, `위험요인`)로 강제 매핑하는 로직을 유지함.

### 7.2 체크리스트(2단계) UI/UX 표준
- **체크박스 인터랙션**: 각 항목 좌측에 **독립적인 체크박스**를 배치하여, 상세 내용을 열지 않고도 즉시 점검(Toggle)이 가능해야 함.
- **아코디언 상세 노출**: 위험요인 제목 영역 클릭 시에만 상세 조치 내역과 위험도 정보가 확장됨.
- **위험도 스코어보드**: 상세 영역 내에 **[현재 빈도/강도/위험성]** 및 **[개선 빈도/강도/위험성]** 수치를 배지(Badge) 형태로 명시하여 시각적 직관성을 제공함.

### 7.3 환경 정돈 (Housekeeping)
- **임시 파일 제거**: 데이터 복구 또는 정제를 위해 생성한 일회성 스크립트(`scratch/*.js` 등)는 작업 완료 후 즉시 자가 삭제하여 워크스페이스의 청결도를 유지함.

### 8. 비즈니스 로직 및 UI/UX 표준 (v35.4.4)

#### 8.1 Gap-Analysis 자동화 로직
- **정의**: '현재 안전조치'에서 체크되지 않은 항목은 하단 '개선대책' 섹션으로 자동 전이됨.
- **구현**: `smartSplit` 결과를 기반으로 `currentState.checkedMeasures` 포함 여부를 실시간 검사함.

#### 8.2 개선대책 체크박스 및 레드 테마
- **체크박스**: 개선대책 리스트의 각 항목 역시 개별 체크박스를 보유해야 함.
- **스타일링**: 
  - **미체크 상태**: 반드시 **강렬한 빨간색 테두리와 배경**(`#fef2f2`, `#fca5a5`)을 적용하여 조치 필요성을 시각화함.
  - **체크 완료 상태**: 녹색 또는 파란색 테마로 전환하여 조치 확정을 표시함.

#### 8.3 위험도 평가 보드 (인터랙션 표준)
- **입력 방식**: 버튼 나열 방식이 아닌 **드롭다운(Select Box)** 방식을 사용함.
- **레이아웃**: `빈도(L) | 강도(S) | 위험도(R)`의 세 항목을 **한 줄(Flex Row)**에 수평 정렬함.
- **실시간 연동**: 드롭다운 변경 즉시 `R = L × S` 연산을 수행하고 `currentState.customRiskScores`에 저장함.
