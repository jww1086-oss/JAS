# 🧠 발전소 위험성평가 시스템 통합 메모리 (v35.8.14)

이 문서는 시스템의 핵심 지능과 무형의 자산을 보존하는 '기억 장치'입니다.

## 1. 🎨 디자인 유전자 (UI/UX Memory)
- **Flat Philosophy**: 박스 중심에서 리스트 중심으로 전환하여 공간 효율성 100% 확보.
- **Word Integrity**: `keep-all`을 통한 한글 단어 보존 줄바꿈 표준화.
- **Step Branding**: 28px 고정 크기의 단계 숫자와 상단 정렬 레이아웃.

## 2. 🧠 지능형 로직 (Logic Memory)
- **Auto-Deduplication**: 위험요인 문구가 조치 사항에 반복될 경우 지능적으로 필터링.
- **Dynamic Transition**: 미이행 조치를 개선 대책으로 자동 전이하는 Gap-Analysis 엔진.
- **Real-time Scoring**: 빈도(L) × 강도(S) = 위험도(R)의 실시간 상태 관리.

## 3. ⚙️ 운영 기술 (Technical Memory)
- **Cache Shield**: 배포 시마다 버전 쿼리를 갱신하여 클라이언트 캐시 오염 방지.
- **State Sync**: `syncUIState`를 통한 단계별 헤더 및 배너 동적 제어.

## 4. 🚀 사용자 여정 (Flow Memory)
1. **Selection**: 부서 ➔ 작업 단위 필터링.
2. **Evaluation**: 이행 내역 체크 ➔ 위험도 평가 ➔ 개선 대책 전이.
3. **Drafting**: 실시간 자동 저장(Draft) 및 복원 시스템.
4. **Finalize**: 리포트 요약 ➔ 전자 서명 ➔ 데이터 전송.
