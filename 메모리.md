# 🧠 KOMIPO 프로젝트 작업 메모리 (v35.4.1 기준)

이 문서는 시스템 복구 과정에서의 핵심 결정 사항과 기술적 해결책을 보존하여 지식의 연속성을 보장합니다.

## 1. 프로젝트 개요
- **목적**: KOMIPO 스마트 안전 점검 시스템 데이터 표준화 및 UI/UX 정상화
- **데이터 소스**: 구글 시트 원본 (2,440행) 기반 `master_data.json`

## 2. 핵심 표준 (Standard)
- **한글 인코딩**: 100% UTF-8 무결성 유지 (깨짐 방지)
- **계층 구조**: 부서명(Level 1) → 작업명(Level 2) → 작업단계(Level 3)
- **데이터 필드**: 구글 시트 12개 헤더와 1:1 매핑 유지

## 3. 주요 트러블슈팅 및 해결책
### A. 브라우저 캐싱 문제 (가장 큰 난관)
- **현상**: 코드를 수정해도 브라우저가 예전 버전을 계속 실행함.
- **해결**: 모든 리소스 호출부에 버전 파라미터(`?v=35.4.1`)를 추가하고, 데이터 페치 시 타임스탬프(`?t=Date.now()`)를 적용하여 캐시를 무력화함.

### B. 로딩 동기화 (레이스 컨디션)
- **현상**: 데이터가 다 읽히기도 전에 UI 배너가 그려져 빈 화면(스켈레톤)이 노출됨.
- **해결**: `init()` 함수 내에서 데이터 인덱싱이 100% 완료된 후에만 로딩 오버레이를 제거하고 화면을 전환하도록 로직 동기화.

### C. 구문 오류 복구 (v35.4.1)
- **현상**: `app.js` 내 중복된 `catch` 블록으로 인해 스크립트 실행이 중단되어 UI 응답이 없었음.
- **해결**: 중복 블록 제거 및 로직 정제(selectTask 일원화)를 통해 시스템 안정성 확보.

- `app.js`: 핵심 비동기 로딩 및 내비게이션 엔진 (v35.4.1)
