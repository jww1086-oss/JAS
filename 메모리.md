## 🛠 핵심 시스템 로직 (v34.x 패치)

### 1. 데이터 수치 정합성 보정 (Master Data Fallback)
- **문제**: 드롭다운 선택 시 빈도·강도 수치가 `1`로 초기화되거나 누락되는 현상.
- **해결**: `updateRiskScoreByHash` 및 `submitLog` 단계에서 `currentState.risks`(구글 시트 원본 데이터)를 우선적으로 대조하여, 누락된 수치를 강제 복구함.
- **주의**: 시트 헤더명이 `현재_빈도`, `현재_강도` 등 한글로 되어 있음을 항상 고려해야 함.

### 2. 지능형 평가자 유효성 검사 (Multi-Worker Tags)
- **구조**: 단일 `selectedWorker` 변수 대신 `selectedWorkers` 배열(태그 형식)을 주력으로 사용.
- **로직**: 이륙 전(`nextStep`) 검사 시 태그 리스트에 인원이 있다면 입력창이 비어있어도 통과시키며, 입력창에만 이름이 있는 경우 자동으로 태그에 추가하고 진행함.

### 3. 실시간 동기화 및 캐시 무력화 (Anti-Confusion Sync)
- **SYNC**: 데이터를 가져올 때 `?t=timestamp` 쿼리 스트링을 사용하여 브라우저 및 구글 서버 캐시를 원천 차단함.
- **정화**: 과거의 테스트용 샘플 데이터(계전기술부 등) 주입 로직을 완전히 삭제하여, 오직 구글 시트의 실제 데이터만 투명하게 노출함.

### 4. 보고서 UI 폴리싱
- **(메모) 삭제**: 수기 입력 개선대책 앞에 붙던 거추장스러운 접두사를 제거하여 전문적인 보고서 형식 유지.

## 📝 유지보수 가이드 (향후 수정 시 필독)
